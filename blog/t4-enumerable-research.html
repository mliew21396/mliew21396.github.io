Enumerable Discovery - grep

July 7th, 2015

Of the many popular Enumerable methods, the ones that come to mind are "map" and "select." Though these are the most popular, they are not necessarily the most powerful. In this post, I will discover the intricacies of "grep" and unveil the possibilites of this method.

Below is the basic format from the ruby docs.

enum.grep(pattern) { |obj| block} => array

The above returns an array of every element in enum for which the pattern === element. If the optional block is applied, each matching element is passed through it, and the block's result is stored in the output array.

Grep can act as a simpler select by filtering by pattern matching rather than giving a block to "select." To do this case with "select", RegExp would be needed. Instead, you can simply choose a pattern with grep:


          [1,10,100,1000].grep(1..100) # => [1, 10, 100]
          [1,'a',2,'b'].grep(Integer) # => [1,2]

This filtering is easy to customize. You can even use "==" to throw a single condition as a filter.

The power of the "map" method is bulk transforming and grep can does this too.


        [1,2,3,4].grep(1..10){|x| x + 1}

This will add 1 to each of the integers. As you may have noticed, this also does the select functionality and limits the array to a range of 1..10. Two birds with one stone! This style of coding can be called "selective bulk transformation." Very useful!

I will briefly say that grep can be utilized even more with RegExp, but that's for another post. Hope this help you implement grep as a more formidable tool.
