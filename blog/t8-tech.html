Exploring Regular Expressions

August 6th, 2015

Regular Expressions, aka Regex, can be a very powerful tool in web development ranging from aiding validations to creating better search/select algorithms. Specifically, it can be used to narrow down a plethora of conditionals to a mere line or so of seemingly gibberish.

The power of Regex can be seen by example. Say you need to write a conditional searching for strings that match a group of image files range from 1 to 9. They are as so - "Image01", "Image02", "Image03", and so on. Writing this by hand would be pretty long: 


string = "Image0" + (rand()*9+1).floor.to_s # randomizing string
p string # always set to Image01 through Image09
if (string == "Image01" || string == "Image02"... string == "Image09")
puts "Match!"
end

# returns Match!
That's quite lengthy. With Regex, it would instead be much shorter(note that when comparing strings with regex, the double equal sign is instead replaced with =~). 

string = "Image0" + (rand()*9+1).floor.to_s # randomizing string
p string # always set to Image01 through Image09
if (string =~ /Image0[1-9]/)
puts "Match!"
end

# returns Match!
This is much shorter! In short of what's happening, a forward slash begins and ends the regex comparison. Anything typed in between these forward slashes our searched for as written. As with everything, there are exceptions that show the power of regex. The brackets indicate a range to search for for that character. In this case, the range is between 1 and 9.
